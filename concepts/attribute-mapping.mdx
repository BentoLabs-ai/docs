The BentoLabs SDK is a thin layer over OpenTelemetry. Each kwarg you pass to `track_ai` (or set on a trajectory) becomes one OTel span attribute, which our ingest first-classes into a column. **Anything you don't pass becomes invisible to that column.**

## Built-in fields

| Kwarg | OTel attribute | Lands in | Skip it and... |
|---|---|---|---|
| `event` (required) | `span.name` | `spans.name` | (required) |
| `user_id` | `gen_ai.user.id` | `traces.user_id` | No user filter in the UI |
| `convo_id` | `gen_ai.conversation.id` | `traces.session_id` | Conversations don't link, every turn looks standalone |
| `model` | `gen_ai.request.model` | `spans.model` | No model breakdown or cost view |
| `provider` | `gen_ai.system` | `spans.provider` | No provider tag in lists or filters |
| `input` | `input.value` | `spans.input` | Empty input column |
| `output` | `output.value` | `spans.output` | Empty output column |
| `properties` | each key as-is | `spans.attributes` JSONB | Custom dimensions missing |

## Why you should pass all four of these

If you emit one of these and not the others, you lose dashboard features for no reason.

<AccordionGroup>
  <Accordion title="provider (no auto-inference)">
    A model name like `claude-3-5-sonnet` does NOT auto-tag as Anthropic. Bedrock model IDs (`anthropic.claude-3-...`) are ambiguous on purpose. Pass it explicitly: `"openai"`, `"anthropic"`, `"google"`, `"aws_bedrock"`, etc.
  </Accordion>

  <Accordion title="convo_id (without this, conversations fragment)">
    Without `convo_id`, multi-turn conversations look like N independent requests. The conversation timeline view is empty.
  </Accordion>

  <Accordion title="user_id (required for user filtering)">
    Required to filter the trace list by user. We do not store any profile data (no email, no name, no traits). `user_id` is a pass-through string only.
  </Accordion>

  <Accordion title="model (required for cost view)">
    Required for the cost view and per-model breakdowns. Without it, spend rolls up under "Unknown."
  </Accordion>
</AccordionGroup>

## Property type fidelity

When you pass `properties=...`, values keep their type so downstream filters and aggregates work correctly.

| You pass | Span sees | Why it matters |
|---|---|---|
| `"foo"` (str) | string |   |
| `42` (int) | int | dashboard can `experiment_id > 100` |
| `3.14` (float) | float |   |
| `True` (bool) | bool | dashboard can `is_premium = true` |
| `[1, 2, 3]` (homogeneous list) | array |   |
| `{"nested": "x"}` or `[1, "two"]` (dict / mixed) | JSON string (fallback) | OTel doesn't natively support these |

<Warning>
  Don't put `gen_ai.*`, `input.value`, or `output.value` in `properties`. The SDK-managed kwargs are written *after* properties and will overwrite them. Use the dedicated kwargs instead.
</Warning>

## Common mistakes

1. **Forgetting `provider`.** Half your dashboard filters go unused. Pass it on every call.
2. **Forgetting `convo_id`.** Conversations look fragmented. Every multi-turn call needs the same ID.
3. **Setting `gen_ai.*` keys in `properties`.** They get silently overwritten by the SDK kwargs.
4. **No `flush()` in short scripts.** The exporter batches. On `os._exit()` or a quick termination, the last batch is dropped. Always call `bento.flush()` (or `bento.shutdown()` which flushes too) before a short-lived process exits.
5. **Bedrock model IDs.** `model="anthropic.claude-3-sonnet-20240229-v1:0"` needs `provider="aws_bedrock"`, not `"anthropic"`.
