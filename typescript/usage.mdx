## Span processor

```ts
import { BentoLabsSpanProcessor } from "@bentolabs/sdk";
import { NodeTracerProvider } from "@opentelemetry/sdk-trace-node";

const provider = new NodeTracerProvider({
  spanProcessors: [new BentoLabsSpanProcessor()],
});
provider.register();

// record spans via @opentelemetry/api or any OTel-instrumented library

await provider.forceFlush();
await provider.shutdown();
```

`BentoLabsSpanProcessor` reads `BENTOLABS_API_KEY` and `BENTOLABS_BASE_URL` from `process.env` by default. Pass `{ apiKey, baseUrl }` to override from code:

```ts
new BentoLabsSpanProcessor({
  apiKey: "bl_pk_...",
  baseUrl: "http://localhost:8080",
});
```

The processor batches spans with OTel SDK defaults and POSTs them to `${baseUrl}/v1/traces` with `Authorization: Bearer bl_pk_...`.

## Setting attributes

To get the same dashboard columns as the Python analytics layer, set these OTel attributes on your spans:

```ts
import { trace } from "@opentelemetry/api";

const tracer = trace.getTracer("my-app");

tracer.startActiveSpan("user_message", (span) => {
  span.setAttribute("gen_ai.user.id", "user_42");
  span.setAttribute("gen_ai.conversation.id", "conv_abc");
  span.setAttribute("gen_ai.request.model", "claude-3-5-sonnet-20241022");
  span.setAttribute("gen_ai.system", "anthropic");
  span.setAttribute("input.value", "What's the capital of France?");
  span.setAttribute("output.value", "Paris.");
  span.end();
});
```

See [attribute mapping](/concepts/attribute-mapping) for the full table of which attributes land in which dashboard columns.

<Tip>
  For tool spans, also set `openinference.span.kind = "tool"`. The ingest mapper folds this into `spans.kind` so the monitoring layer's tool detector picks it up.
</Tip>

## Lower level: just the exporter

If you need to wire your own `SpanProcessor` (e.g. `SimpleSpanProcessor` in tests, or a `BatchSpanProcessor` with custom batching limits), use `BentoLabsTraceExporter` directly:

```ts
import { BentoLabsTraceExporter } from "@bentolabs/sdk";
import { SimpleSpanProcessor } from "@opentelemetry/sdk-trace-base";
import { NodeTracerProvider } from "@opentelemetry/sdk-trace-node";

const provider = new NodeTracerProvider({
  spanProcessors: [new SimpleSpanProcessor(new BentoLabsTraceExporter())],
});
```

## Configuration

| Field | Resolution order |
|---|---|
| `apiKey` | `opts.apiKey` â†’ `process.env.BENTOLABS_API_KEY` â†’ throws `BentoAuthError` |
| `baseUrl` | `opts.baseUrl` â†’ `process.env.BENTOLABS_BASE_URL` â†’ `https://api.bentolabs.ai` |

```ts

const cfg = resolveOptions();
// { apiKey: string, baseUrl: string } (frozen)
```

## Errors

`resolveOptions` (called inside `BentoLabsSpanProcessor` and `BentoLabsTraceExporter` constructors) throws `BentoAuthError` for fail-fast cases:

```ts
import { BentoAuthError, resolveOptions } from "@bentolabs/sdk";

try {
  const cfg = resolveOptions();
} catch (err) {
  if (err instanceof BentoAuthError && err.code === "missing_api_key") {
    // surface a graceful "telemetry disabled" path
  }
  throw err;
}
```

| Code | When |
|---|---|
| `missing_api_key` | No key passed and `BENTOLABS_API_KEY` not set. |
| `invalid_api_key_format` | Key does not start with `bl_pk_`. |

Catch it at boot if you want a graceful "no telemetry today" fallback rather than crashing the host.
