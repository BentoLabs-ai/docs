BentoLabs is an observability platform for AI applications. You add one SDK call where your model runs, and the dashboard handles the rest: multi-turn conversation timelines, cost and token breakdowns, tool-call traces, and per-user filtering.

## Why BentoLabs

You probably already have OpenTelemetry instrumentation somewhere. Bento doesn't replace it. The SDK ships as an OTLP span processor under the hood, so it slots into any existing tracer provider without conflicts. If you'd rather not think about OTel at all, the analytics layer (`bento.track_ai`, `bento.begin`, decorators) gives you a Raindrop-shaped API that's three lines of setup.

The platform is opinionated about AI workloads: `gen_ai.*` semantic conventions are first-classed into dashboard columns, conversation IDs link multi-turn flows, and tool spans pick up automatic classification. No user profile storage, no SDK-side PII inference, no auto-tagging that gets things wrong.
